# 包装系统优化规则 (Gemini CLI)

## 1. 核心原则
- **成本核算页保护**：严禁修改 `panel-cost` 的布局、样式及计算规则。该页面必须保持原有形态，确保计算精确性与用户习惯。
- **合理紧凑**：优化后的“新版”前端必须在保持美感的同时，追求信息密度。严禁留白过多，确保用户在单个屏幕内看到尽可能多的核心信息。
- **移动端优先**：所有优化必须深度适配移动端。筛选器、统计汇总、列表项在手机上应采用紧凑布局（如更小的内边距、更紧密的栅格）。
- **双版本并存**：必须保留“经典版”与“现代版”的切换功能。默认给用户经典版，仅在用户主动选择时切换到现代版。

## 2. 模块规范
- **订单管理**：
  - 列表项（Card）应包含核心规格、数量、印膜、压辊、状态等信息。
  - 统计汇总（Summary）在手机端应采用紧密栅格或精简布局，避免占用过多垂直空间。
- **导航框架**：
  - 现代版采用 Glassmorphism 或简洁白底风格。
  - 菜单栏应在手机端自动折叠为侧边栏。

## 3. 开发流程
- **即时推送**：所有有效修改必须及时通过 `git push` 同步到远程仓库。
- **功能测试**：每次修改后必须运行 `npm run verify:smoke` 确保核心链路（登录、开单、流程流转）正常。
- **回归修复**：若测试发现遗漏或 Bug，必须优先修复。
